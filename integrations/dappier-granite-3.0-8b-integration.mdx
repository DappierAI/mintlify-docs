## Dappier Retriever for LangChain

The `DappierRetriever` is a LangChain-compatible component that seamlessly integrates Dappier's real-time data capabilities into your LangChain workflows. It allows your LLM applications, powered by models like IBM Watsonx Granite, to fetch and utilize current information from diverse sources before generating a response or taking an action.

This guide will help you get started with using the `DappierRetriever` in your LangChain and IBM Watsonx Granite projects.

You can also check out a [Cookbook demonstrating a Q&A system](https://colab.research.google.com/drive/1VV8VzUTK8YFCeuiBZbXTJ5oIQce43ovP?usp=sharing) using LangChain, Dappier, and IBM Watsonx Granite.

## DappierRetriever Parameters

The DappierRetriever accepts several parameters to customize its behavior:
### `data_model_id` (str):

    The Dappier data model ID to query against. Data model IDs define the scope of the data (e.g., general web, financial news, sports, weather).

    Data model IDs always start with the prefix "dm_".

    You can find available data model IDs at the Dappier Marketplace.

    Default: If not specified, Dappier might use a general-purpose model, but explicitly setting it is recommended for targeted results.

### `k` (int, optional):

    The number of documents to retrieve.

    Default: 9 (as per Dappier's API defaults for some underlying calls, but can vary. It's good practice to set it explicitly).

### `search_algorithm` (Literal["most_recent", "most_recent_semantic", "semantic", "trending"], optional):

    The search algorithm to use for retrieving articles.

        "most_recent": Prioritizes the newest content.

        "semantic": Prioritizes conceptual relevance to the query.

        "most_recent_semantic": Balances recency and semantic relevance.

        "trending": Focuses on currently popular or trending content within the data model.

    Default: Dappier's default, often "most_recent". Check Dappier documentation for specifics per data model.

### `ref` (str, optional):

    A site domain (e.g., "example.com") to prioritize or filter results from. The exact behavior (prioritization vs. strict filtering) may depend on the Dappier data model and API.

    Default: None.

### `num_articles_ref` (int, optional):

    When ref is specified, this suggests the minimum number of articles to be sourced from the ref domain. The retriever will attempt to fulfill this, with the rest coming from other sources within the data model.

    Default: 0.

### `api_key` (str, optional):

    Your Dappier API key. If not provided directly, the retriever will attempt to load it from the DAPPIER_API_KEY environment variable.

    Default: None (expects environment variable).
