**Title:** ⚡️ Real-Time Q&A with LangChain, Dappier & IBM Watsonx Granite: Powering Answers with Fresh Data

**Introduction**

This integration guide demonstrates how to build a powerful Question-Answering (Q&A) system leveraging **LangChain** to orchestrate **Dappier**'s real-time data retrieval with the robust reasoning capabilities of **IBM Watsonx AI's Granite** model. The result is an application capable of providing answers grounded in the most current information available.

In this guide, you'll explore:

*   **LangChain**: The framework used to chain together data retrieval, prompting, and language model inference for seamless RAG execution.
*   **Dappier**: A platform connecting LLMs to real-time, rights-cleared data. We utilize its `DappierRetriever` for LangChain to fetch contextually relevant, up-to-the-minute data based on user queries.
*   **IBM Watsonx AI**: An enterprise-grade AI platform. We feature the `ibm/granite-13b-instruct-v2` model for its strong instruction-following and generative text capabilities.
*   **Retrieval-Augmented Generation (RAG)**: The core pattern enabling the LLM to generate informed responses based on freshly retrieved data, minimizing hallucinations and ensuring relevance.

This setup provides a foundation for building dynamic applications like market analysis tools, news summarizers, or intelligent FAQ systems that require access to current information.

## ⚙️ Configuring Core Components

With credentials set up, we configure the Dappier Retriever and the Watsonx LLM within LangChain.

### Step 1: Configure Dappier Retriever

The `DappierRetriever` fetches relevant documents from Dappier's real-time data sources based on the user query.

*   `data_model_id`: Specifies the Dappier data source (e.g., news, finance). Find IDs on the [Dappier Marketplace](https://platform.dappier.com/marketplace).
*   `k`: Determines the number of documents to retrieve.

```python
from langchain_dappier import DappierRetriever

# Example configuration using a general web/news data model
retriever = DappierRetriever(
    data_model_id="dm_01jagy9nqaeer9hxx8z1sk1jx6", # Target data source
    k=3  # Number of documents to fetch
    # api_key=dappier_api_key # Can be passed directly instead of env var
)
```

> 🛰️ The retriever is now configured to fetch the top 3 relevant documents from the specified Dappier data model for any given query.

### Step 2: Configure IBM Watsonx Granite LLM

Instantiate the Watsonx LLM, specifying the model, credentials, and generation parameters.

*   `model_id`: We use `ibm/granite-13b-instruct-v2` for its instruction-following capabilities.
*   `params`: Control text generation (e.g., `DECODING_METHOD`, `TEMPERATURE`, `MAX_NEW_TOKENS`). `greedy` decoding with `temperature=0` provides deterministic outputs suitable for Q&A.

```python
from langchain_ibm import WatsonxLLM
from ibm_watson_machine_learning.metanames import GenTextParamsMetaNames as GenParams

# Prepare credentials dictionary
credentials = {
    "url": f"https://{os.environ['IC_REGION']}.ml.cloud.ibm.com",
    "apikey": os.environ["WATSONX_APIKEY"],
}

# Define generation parameters for controlled output
params = {
    GenParams.DECODING_METHOD: "greedy",
    GenParams.TEMPERATURE: 0,
    GenParams.MIN_NEW_TOKENS: 10,
    GenParams.MAX_NEW_TOKENS: 300,
    GenParams.STOP_SEQUENCES: ["Human:", "Observation:"], # Optional stop sequences
}

# Instantiate the LLM
llm = WatsonxLLM(
    model_id="ibm/granite-13b-instruct-v2",
    url=credentials.get("url"),
    apikey=credentials.get("apikey"),
    project_id=os.environ["PROJECT_ID"],
    params=params,
)
```

> 🤖 The Watsonx Granite model is now ready to receive prompts and generate responses based on the provided context.

## 🔗 Building the RAG Chain with LangChain

We use LangChain Expression Language (LCEL) to define the flow: retrieve documents, format context, prompt the LLM, and parse the output.

### Step 1: Define the Prompt Template

Instruct the LLM on how to use the retrieved context to answer the question.

```python
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template(
    """Answer the following question based *only* on the context provided below. If the context doesn't contain the answer, state that clearly.

Context:
{context}

Question: {question}"""
)
```

### Step 2: Construct the Chain

Combine the components using LCEL's pipe (`|`) operator.

```python
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# Helper function to format documents (included in the notebook)
def format_docs(docs):
    return "\n\n---\n\n".join(doc.page_content for doc in docs)

# Define the RAG chain
chain = (
    {
        "context": retriever | format_docs,  # Retrieve & format context
        "question": RunnablePassthrough() # Pass original question
    }
    | prompt          # Populate prompt
    | llm             # Send to Watsonx LLM
    | StrOutputParser() # Get string output
)
```

> ⛓️ This chain encapsulates the entire RAG process: query -> retrieve -> format -> prompt -> generate -> parse.

## 🚀 Running the Q&A System

With the chain built, you can invoke it with a user query to get real-time answers.

### Step 1: Define a Query

Formulate a question for which you need up-to-date information.

```python
user_query = "What were the main announcements from Apple's latest event?"
```

### Step 2: Invoke the Chain

Pass the query to the chain's `invoke` method.

```python
# The chain takes the query string directly
response = chain.invoke(user_query)

print("\nGenerated Answer:")
print(response)
```

The output synthesizes information from the real-time documents fetched by Dappier. The exact answer will vary based on the data available at the time of execution.

The event also showcased updates to macOS Sequoia and watchOS 11.
